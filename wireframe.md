# Wireframe — 4 Tab 草圖（文字版）

> 手機優先設計（375px 寬基準），桌面響應式擴展。
> **使用者：胰島素幫浦使用者，所有劑量 0.1U 精度，顯示 `X.X U` 格式。**

---

## 通用元件（所有 tab 都有）

```
┌──────────────────────────────────┐
│  🍠 醣類計算器          ↻ 🌙    │  ← 頂部 bar：app 名 + refresh + 暗色切換
├──────────────────────────────────┤
│                                  │
│        （tab 內容區）            │
│                                  │
├──────────────────────────────────┤
│  🧮 計算  📋 紀錄  📈 趨勢  ⚙️  │  ← 底部 tab bar（手機）
└──────────────────────────────────┘
```

**色系：黃 + 紅 + 橘 + 白暖色組合（不是純黃）**

- 頁面底色：暖米白 `#FFFAF0`（亮）/ 暖深棕黑 `#1F1610`（暗）
- 頂部 bar 背景：飽和黃 `#FFC107`，文字暖深棕 `#3D2914`
- 底部 tab bar：白底（亮）/ 暖深棕（暗），active tab = 飽和黃 underline + 暖深棕字
- 卡片：白 `#FFFFFF`，邊框暖淺金 `#F4E4BC`
- 內文/數字：暖深棕 `#3D2914`（不用純黑——配黃會刺眼）

## 互動細節

- **手勢**：列表項向左滑 = 刪除（要確認）/ 向右滑 = 編輯
- **長按**：食材 chip 長按 = 編輯該食材的營養值
- **快捷**：計算器頁右下角浮動「+ 紀錄」按鈕（直接從目前計算結果開 form）
- **空狀態**：
  - 紀錄 tab 沒資料 → 「還沒有紀錄。從計算器開始第一餐 →」
  - 趨勢 tab 沒資料 → 「需要至少 3 筆紀錄才有趨勢」

## 餐名自動建議規則摘要

新增紀錄時，餐名 input 預填按以下優先順序：

1. **照片匹配**（pHash d<8）→ 帶出已存紀錄的餐名
2. **食材匹配**（食材集合相同、克數差 <10%）→ 帶出已存紀錄的餐名
3. **自動 compose**：取碳水佔比最高的前 3 個食材，例「白米飯 + 雞胸 + 花椰」
4. **退路**：沒食材時用「早餐 0501 / 午餐 0501 / 晚餐 0501」

來源 icon：
- 📷 = 照片匹配
- ✓ = 食材匹配
- 🆕 = 自動產生（無匹配）

input 提示：淡灰底文字「建議名，點可修改」。使用者點 → 進編輯模式。
儲存後若同名 → 歸到同一 series。

## 劑量輸入細節（幫浦 0.1U）

- 劑量輸入欄：HTML `<input type="number" step="0.1" min="0">`，跳手機數字鍵盤含小數點
- 快速增減按鈕：[−0.1] [−0.5] [+0.1] [+0.5]，長按連續觸發
- 顯示永遠 `X.X U`（toFixed(1)），整數補 `.0`
- 「最佳劑量」結果四捨五入到 0.1U（不是 0.05U）
- I:C ratio 顯示 `1:X.X`（例 1:9.4），分母 round 到 0.1

| 元素 | 亮模式 | 暗模式 |
|---|---|---|
| 頁面底 | 暖米白 `#FFFAF0` | 暖深棕黑 `#1F1610` |
| 頂部 bar | 飽和黃 `#FFC107` | 暖黃 `#FFB300`（降彩度） |
| 卡片 | 白 `#FFFFFF` | 暖深 `#2A1E16` |
| 邊框 | 暖淺金 `#F4E4BC` | 暖深棕 `#5C4A3A` |
| 內文/數字 | 暖深棕 `#3D2914` | 暖淺米 `#F5E6D3` |
| 警示紅 | `#E53935` | `#E53935`（同） |
| 警示橘 | `#FB8C00` | `#FB8C00`（同） |

警示色（紅/橘）兩模式同色——飽和度本來就夠，確保可讀。
